# shuffle_psv

`shuffle_psv` は、PSV ファイル内のレコードをランダムにシャッフルするツールです。

- 入力: PackedSfenValue 40 バイト固定長ファイル
- 出力: 同一レコード群をランダム順に並べ替えたファイル
- 特徴: シード指定で再現可能、大規模ファイル対応のチャンク方式あり

NNUE 学習では教師データの並び順がバッチ勾配に影響するため、十分なシャッフルが重要です。

## シャッフル方式

### インメモリ方式（デフォルト）

全レコードをメモリに読み込み、Fisher-Yates アルゴリズムで完全シャッフルします。

- メモリ使用量 = ファイルサイズ
- 最も高品質なシャッフル
- メモリに収まるファイルサイズでのみ使用可能

### チャンク方式（`--chunk-size` 指定時）

メモリに載りきらない大規模ファイル用。2 パスで処理します。

**Pass 1**: 各レコードをランダムに選んだチャンクファイルに振り分け

- レコードの所属チャンクがランダムに決まるため、チャンク間のシャッフルが実現される

**Pass 2**: 各チャンクをメモリに読み込み、チャンク内を Fisher-Yates でシャッフルし、出力ファイルに書き出し

- 複数チャンクをバッチ並列で処理（利用可能メモリの 70% を上限に自動制限）
- 一時ファイルは `TMPDIR` 環境変数で指定したディレクトリに作成される

結果として、チャンク間・チャンク内ともにランダムな完全シャッフルになります。

## チャンクサイズの選び方

`--chunk-size` はチャンクあたりの **レコード数** で指定します。

メモリ使用量の目安: `chunk_size × 40 bytes`

| chunk_size | メモリ使用量/チャンク |
|---|---|
| 500,000,000 (5億) | 約 18.6 GiB |
| 1,000,000,000 (10億) | 約 37.3 GiB |
| 2,000,000,000 (20億) | 約 74.5 GiB |

Pass 2 では複数チャンクを並列ロードするため、実際のピークメモリは上記の倍数になります。バッチサイズは利用可能メモリに基づいて自動制限されます。

チャンク数は最大 1,000 に制限されています。チャンクが多すぎる場合は `--chunk-size` を増やしてください。

## 一時ファイルのディスク使用

チャンク方式では、Pass 1 で入力ファイルと同じサイズの一時ファイルが作成されます。

- デフォルトの一時ディレクトリは `/tmp`（システムの `TMPDIR` に従う）
- 大規模ファイルの場合、`/tmp` の空き容量が不足する可能性がある
- `TMPDIR` 環境変数で十分な空き容量のあるディレクトリを指定すること

```bash
# 例: 一時ファイルを /mnt/nvme1/tmp に作成
TMPDIR=/mnt/nvme1/tmp shuffle_psv --input large.psv --output shuffled.psv --chunk-size 1000000000
```

## 使用例

### 基本: 小規模ファイルのシャッフル

```bash
cargo run --release --bin shuffle_psv -- \
  --input data.psv --output shuffled.psv
```

### シード指定（再現性のため）

```bash
cargo run --release --bin shuffle_psv -- \
  --input data.psv --output shuffled.psv --seed 42
```

### 大規模ファイル: チャンク方式

```bash
TMPDIR=/mnt/nvme1/tmp cargo run --release --bin shuffle_psv -- \
  --input large.psv --output shuffled.psv \
  --chunk-size 1000000000 --seed 42
```

## オプション一覧

| オプション | 説明 | デフォルト |
|---|---|---|
| `--input`, `-i` | 入力 PSV ファイルパス | — |
| `--output`, `-o` | 出力 PSV ファイルパス | — |
| `--seed` | 乱数シード（再現性のため） | ランダム |
| `--chunk-size` | チャンクサイズ（レコード数）。0 = インメモリ方式 | `0` |

## 注意

- 入力ファイルは変更しません
- 出力ファイルのレコード数は入力と同一です（増減なし）
- Ctrl-C で中断可能（出力は不完全になります）
- チャンク方式では一時ディスクが入力ファイルと同量必要です
