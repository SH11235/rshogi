# 変更履歴

このファイルには、プロジェクトの重要な変更がすべて記録されています。

フォーマットは [Keep a Changelog](https://keepachangelog.com/ja/1.0.0/) に基づいており、
このプロジェクトは [セマンティック バージョニング](https://semver.org/lang/ja/) に準拠しています。

## [未リリース]

### 追加
- Phase 3 Lazy SMP並列探索の実装
  - 動的閾値フラッシュ機能付きローカルノードカウンター
  - CachePaddedによるフォルスシェアリング対策
  - 重複作業削減のためのABDADA-lite実装
  - 包括的な並列探索ベンチマークスイート
- 時間管理システムの強化
  - 停止チェック頻度の改善
  - 時間切迫状況でのより良い処理
  - 異なるスレッド数での一貫した動作
- セキュリティの改善
  - Gitコマンド実行時に`env_clear()`を追加
  - 空のベンチマーク結果に対する安全なエラー処理

### 変更
- rustdoc用のドキュメントフォーマットを改善
- 一時的な開発ドキュメントを統合
- 並列探索の将来改善ドキュメントを更新

### 修正
- 空のベンチマーク結果でのパニックの可能性を修正
- measure_stop_latency関数の浮動小数点精度問題を修正
- スレッド数解析のエラーメッセージを改善
- rustdocの壊れた内部ドキュメントリンクを修正

### パフォーマンス
- 4スレッドで2.5倍の高速化を達成（目標: 2.4倍）
- 重複率を30%未満に削減（目標: 35%以下）
- 98%以上のPV一致率を維持（目標: 97%以上）
- 停止遅延を一貫して5ms未満に（目標: 5ms以下）

## [0.1.0] - 以前のリリース

### 初期機能
- 基本的な将棋エンジンの実装
- USIプロトコルサポート
- NNUE評価関数
- シングルスレッドのアルファベータ探索
- 置換表
- 定跡サポート
- WASMコンパイルサポート
